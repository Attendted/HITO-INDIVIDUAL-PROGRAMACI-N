PROGRAMACIÓN Y SUS DIFERENCIAS ENTRE SUS TIPOS

La programación orientada a objetos (POO) se basa en objetos que encapsulan datos y comportamiento, mientras que la programación orientada a eventos (POE) se centra en la detección y el manejo de eventos. La programación procedimental se enfoca en la secuencia de acciones y procedimientos. Cada paradigma tiene sus fortalezas y debilidades, y la elección del paradigma depende del problema a resolver y de las necesidades del proyecto.

PROGRMACIÓN ORIENTADA A OBJETOS: Este tipo de programación se podría decir que es es un paradigma de programación que se basa en el concepto de objetos, que son entidades que encapsulan datos y comportamiento. En la POO, también llamada de esta forma (Abreviatura), se utilizan clases para definir la estructura y el comportamiento de los objetos. Las clases son plantillas que definen los atributos y los métodos que los objetos pueden tener. Los objetos se crean a partir de estas clases y se pueden utilizar para realizar diferentes tareas. Principalmente se centra en el concepto de la encapsulación, la herencia y el polimorfismo, lo que permite crear programas más flexibles y escalables.

PROGRAMACIÓN ORIENTADA A EVENTOS: Este tipo de programación también es un paradigma de programación que se centra en la detección y el manejo de eventos, como bien indica su nombre. El evento es una acción que esta en el sistema, como hacer clic en un botón o pulsar una tecla. En la POE (abreviatura), se definen los eventos y se establecen los manejadores de eventos que se ejecutan cuando se produce un evento. Los manejadores de eventos son funciones que se ejecutan en respuesta a un evento específico y pueden realizar diferentes tareas. Esta se utiliza principalmente para el desarrollo de interfaces gráficas de usuario y otros sistemas interactivos.

PROGRAMACIÓN PROCEDIMENTAL: Este tipo de programación se centra en las acciones que se encargan de los probelmas y errores. Se divide el programa en pequeñas unidades de código llamadas procedimientos o funciones, que realizan tareas específicas. Estos procedimientos se llaman en una secuencia determinada para realizar una tarea más compleja. Esta se centra en el uso de variables y estructuras de control de flujo para controlar la secuencia de ejecución del programa.
